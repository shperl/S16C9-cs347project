Plan Ownership

In a secure Oracle NoSQL Database deployment, each plan command is associated with its creator as the owner. Only the owner of a plan can see and operate it. If a plan is created in an earlier version of Oracle NoSQL Database or in a non secure store, the owner is null.

Note

The SYSOPER privilege allows a role to perform cancel, execute, interrupt, and wait on any plan.

Users with the SYSVIEW privilege can see plans owned by other users, plans with null owner, and those plans whose owners have been removed from Oracle NoSQL Database.

For more information on configuring Oracle NoSQL Database securely and on roles, see the Oracle NoSQL Database Security Guide.
